# Audit vs "System Deep Dive Analysis" — 2026-07-12

Independent line-by-line verification of the deep-dive report against the actual code, plus the cleanup performed. Verified by `tsc --noEmit` (clean) and a full `next build` (green, all 23 routes + proxy middleware).

## Report claims — verdicts

| Report claim | Verdict |
|---|---|
| EIP-3009 gate in `src/proxy.ts`, off-chain signature verification | **True** |
| "Submits the blockchain transaction asynchronously … very low latency" | **False.** The proxy `waitForTransactionReceipt`s BEFORE serving — settlement is synchronous (~2s+/query). That's a sound fail-closed choice, but the opposite of what the report says. |
| Settlement key is `WALLET_PRIVATE_KEY` | **Wrong name.** It's `SETTLEMENT_PRIVATE_KEY` (and it was missing from `.env.example` — fixed). |
| "Payment replays … system's security is exceptionally strong here" | **Was broken.** The replay check queried `tx_ref` for the client *signature*, but the route stored the *tx hash* there — the check could never match. Every replayed signature reached the chain and burned settlement gas until revert (griefing vector). Fixed: migration `0005` adds a unique `payment_sig` column; proxy checks it, route stores it. Also added off-chain `validBefore`/`validAfter` checks so expired authorizations don't cost gas either. |
| Zod + Ajv fail-closed validation, curation edits re-validated against `output_schema` | **True** (decisions route, executor, schema_validate hook, JsonEditor). |
| `time_spent_ms` mount timers, queue preloading, optimistic UI + rollback, keyboard bindings | **True**, with one bug: the K (skip back) key did `i - 0`. Fixed. JsonEditor advertised Esc/Ctrl+Enter hotkeys that didn't exist — implemented. |
| `after()` backgrounding of `processRun`, timeout risk on large batches | **True** — accurately described; worker-tier recommendation stands. |
| Rate limiter in-memory, isolate-bound, 60/min | **True**, but it only guards `/api/v1/products` and `/api/v1/preview` (per the M1 plan). The paid query route is economically self-limiting once replay is fixed. |
| "Supabase pooler `aws-0-eu-north-1.pooler.supabase.com:6543` … Prisma/PgBouncer" | **Not in this codebase.** The app uses supabase-js over HTTPS (PostgREST) exclusively — no direct Postgres connections, no Prisma. This section of the report is fabricated relative to the code. |
| `ADMIN_EMAILS` env allowlist, redeploy to change | **True.** |
| Prompt injection: generator prompts are admin-controlled, buyer input never reaches prompts | **True** (resolver whitelists filters; only curators write prompts). |

## Bugs found beyond the report (all fixed)

1. **Demand telemetry export dead** — `export-demand.ts` read `payload.query_params`; events store `payload.filters`. Every query collapsed into one empty bucket.
2. **DPO export silently empty** — `export-dpo.ts` used a PostgREST embed across `curation_decisions.run_id`, which has no FK; the query errored and the error was ignored. Rewrote as a two-step fetch; all export scripts now fail loudly on errors.
3. **Promised URLs 404'd** — every paid response links `docs` and `provenance.methodology`; neither page existed (M1 plan WS4 unfinished). Built `/docs` and `/methodology/[slug]` (renders hashes/schemas from the generator row, never the prompt template).
4. **Landing page dishonesty** — advertised a nonexistent endpoint (`/api/v1/products/personality-profile-library`) and claimed "multi-model consensus" + "cosine similarity" the pipeline doesn't do. Corrected to actual endpoint and actual methodology.
5. **`/api/health` static-optimization trap** — parameterless GET gets prerendered at build; would forever serve a build-time snapshot. Added `force-dynamic`.
6. **Hook diagnostics mislabeled** — `hook.executed` events carry no `entity_id`, so the review card showed the whole run's hook events as if they were per-card. Now aggregated per hook and labeled "Run Hook Diagnostics".
7. **Network default mismatch** — `recordPayment` defaulted to `base-sepolia` while the proxy settles on Base mainnet. Default now `base`.
8. Hardcoded public RPC — `BASE_RPC_URL` env override added (defaults unchanged).
