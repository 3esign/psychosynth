# Seven Development Phases

## Phase 1 — MVP
*   **Goal**: Create the smallest possible product that people pay for.
*   **Deliverables**:
    *   **Internal Lab**: Profile Generator, Scenario Generator, Manual Review, Version Control, Tags, Search, Product Builder.
    *   **Database**: Only 5 entities: Profiles, Scenarios, Biases, Emotions, Products.
    *   **Customer Products**: Launch only 3:
        1.  *Synthetic Personality Library* (API, JSON download, CSV).
        2.  *Behavior Scenario Library* (Negotiation, Trading, Stress, Leadership).
        3.  *Trading Psychology Pack* (Bundle of Profiles, Biases, Scenarios, Emotion patterns).
    *   **Architecture**: Next.js, Supabase, Postgres.
    *   **Storage**: Simple API (Stripe/x402 later).

## Phase 2 — Knowledge Graph
Now stop thinking relational; everything becomes connected:
```
Trait ──> Emotion ──> Bias ──> Decision ──> Behavior ──> Scenario ──> Response
```
Now every node knows every other node. You can query:
> *Show every scenario where [High Neuroticism] + [Loss Aversion] + [Crypto Trading] + [Bear Market]*
This becomes extremely valuable.

## Phase 3 — Intelligence Lab
This becomes your internal operating system.
*   **Features**: AI generation, batch generation, quality scoring, similarity detection, duplicate detection, dataset explorer, prompt templates, visual graph explorer, knowledge diff, approval workflow, version comparison, method editor, generation history, review queues.
*   *This is where your competitive advantage compounds.*

## Phase 4 — Product Factory
Now products are no longer hardcoded. Products become "recipes."
*   **Example**: `Trading Psychology Pack = Profiles (where Risk > 0.8) + Bias (Loss Aversion) + Scenarios (Trading) + Emotion (Fear)`.
*   One click yields a new product.

## Phase 5 — Agent Platform
Now you're no longer selling downloads; you're selling intelligence.
*   Products become APIs with endpoints like `POST /simulate`, `/profile`, `/evaluate`, `/emotion`, `/predict`, `/personality`, `/conversation`, `/scenario`.
*   Every endpoint is pay-per-call via x402. Agents pay, receive JSON, and continue reasoning.

## Phase 6 — Agent Marketplace
Everything becomes discoverable. Instead of humans buying datasets, agents discover:
*   Services, capabilities, benchmarks, evaluators, psychological APIs, simulation APIs, scenario APIs.
*   Solana/Base integrations become meaningful because the intelligence itself is being purchased autonomously.

## Phase 7 — Psychological Intelligence OS
This is the long-term vision.
*   No longer a "dataset company" or an "API company."
*   Become **the operating system for synthetic psychological intelligence**.
*   Other companies, researchers, AI agents, game studios, trading firms, healthcare simulations, and education platforms build on top of your underlying knowledge graph.

# World-Class Design Principles

1.  **Everything is versioned**: Never overwrite data; create new versions.
2.  **Everything is composable**: Products are recipes built from reusable components.
3.  **Everything is traceable**: Every record links back to its generation prompts, curation history, quality scores, and revisions.
4.  **Everything is searchable**: Rich metadata, semantic search, and graph traversal are first-class features.
5.  **Everything is API-first**: The dashboard consumes the same APIs that customers and autonomous agents will use.
6.  **Everything is agent-ready**: Every product is accessible through structured APIs with machine-readable schemas.
7.  **Everything is measurable**: Track usage, quality, customer demand, and data lineage to continuously improve the platform.
