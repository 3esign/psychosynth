# PART A — ORIENTATION

## 1. What we are building

**Psychosynth** sells synthetic psychological data to AI agents over HTTP. An autonomous agent (a trading bot, a simulation framework, a personalization engine) calls our API asking for e.g. "20 personality profiles with high neuroticism and strong loss-aversion", receives an HTTP 402 price quote, pays a few cents in USDC on the Base blockchain via the x402 protocol, and gets structured JSON back — no account, no API key, no human.

The data is generated by LLMs **inside our own web app** (the "Lab"), validated by an automated pipeline, and approved by a human curator. Three properties distinguish the product from anyone else calling an LLM:

1. **Provenance** — every served record traces to the exact prompt, model, and parameters that produced it (hashes recorded at generation time).
2. **Psychometric grounding** — personalities use the academically standard Big Five model with population-realistic distributions, not ad-hoc labels.
3. **The Learning Loop** — the system records every generation, validation result, human curation decision, and buyer query as structured data, so the product's quality and the operators' prompts improve continuously, and the operational history itself becomes sellable datasets later.

Two user types: the **admin/curator** (logs into `/lab`, runs generators, reviews items, publishes products) and **buyer agents** (hit `/api/v1/*`, never log in, pay per request). There is also a minimal **public site** (landing page, product catalog, methodology pages) for humans evaluating the service.

**What "delivered" means (Milestone 1):** a deployed web app where (a) the admin generates and curates ≥300 personality profiles through the Lab UI; (b) an autonomous script with a crypto wallet pays $0.01 USDC on Base mainnet and receives profiles, unassisted; (c) for any served profile, the admin can display its full provenance chain; (d) every operational action taken so far is queryable in the events/decisions tables. Milestones 2–3 (§17–18) extend the same codebase; nothing in M1 is throwaway.

### 1.1 Non-goals for Milestone 1

No MCP server, no SDK packages, no Solana, no on-chain hash publishing, no LLM-judge automation, no pack redemption, no multi-user roles, no scenario/response data (all specified for M2/M3). Do not build ahead; the architecture already reserves their places.

### 1.2 Design constitution (rules that resolve disputes)

1. **Everything is versioned** — never overwrite; bump versions, keep history.
2. **Everything is composable** — products are config rows ("recipes"), never new tables or endpoints.
3. **Everything is traceable** — entity → generation run → generator version → prompt hash → content hash, no gaps.
4. **Everything is API-first** — the Lab UI and buyer agents use the same service layer; the UI has no private path to the DB.
5. **Automatic by default, human by exception** — validation hooks route items; humans review what machines can't clear. Automation levels are *earned* with measured evidence (§18), never assumed.
6. **Reproducibility claims are precise** — LLM generation is *replayable* (all inputs recorded), not byte-identical; only query-time composition (post-M3) is byte-deterministic. Never claim otherwise anywhere.
7. **Honesty** — every payload carries `"synthetic": true`; ToS prohibit passing data off as human-sourced.
8. **Everything is a lesson** — if an action taught us something and left no database row behind, that's a bug.

## 2. Glossary

| Term | Meaning here |
|---|---|
| **Big Five / OCEAN** | The standard academic personality model: five traits — Openness, Conscientiousness, Extraversion, Agreeableness, Neuroticism — each scored 0.0–1.0 in our system. |
| **IPIP-NEO** | A public-domain questionnaire instrument for measuring Big Five; we cite its framing in generation prompts and use published population statistics (mean ≈ 0.5, sd ≈ 0.15 per trait) as realism targets. |
| **MBTI** | Popular 4-letter personality labels (e.g. INTJ). Scientifically weak; we derive it from Big Five as a cosmetic, searchable label only. |
| **Cognitive bias** | Systematic deviation from rational judgment (e.g. Loss Aversion). We maintain a reference table of 20, each linked to profiles with a strength value. |
| **Profile** | One synthetic personality: Big Five scores + summary + decision style + tags + linked biases. The atomic sellable unit. |
| **Generator** | A database row defining how to produce entities: prompt template + parameter schema + output schema + model config + hook chain. Not code. |
| **Run** | One logged execution of a generator (N items). Runs are the provenance backbone. |
| **Hook** | A named validation/processing step (schema check, dedup, routing) executed per generated item, composed per-generator via config. |
| **Curation** | Human review of generated items: approve / reject (with reason code) / edit-then-approve. |
| **Recipe** | JSON config describing which entities a product serves and which query filters buyers may use. |
| **Product** | A row pairing a recipe with pricing; exposed at `/api/v1/query/<slug>`. |
| **Dataset** | A named, frozen, hashed snapshot of entity membership (used from M2 for version-pinning products). |
| **Provenance** | Per-entity record of model, prompt hash, template hash, params, and content hash. |
| **Canonical JSON** | JSON serialized with recursively sorted keys and no whitespace — the only form we ever hash. |
| **Events spine** | Append-only `events` table receiving a row for every meaningful action; the Learning Loop's raw material. |
| **Learning Loop** | The subsystem turning operations into improvement: decisions become labels, edits become preference pairs, buyer queries become demand signals, prompt changes become measured experiments. |
| **SFT / DPO** | Fine-tuning data formats: SFT = prompt→approved-output pairs; DPO = (prompt, better, worse) preference triples. Our curation naturally produces both. |
| **x402** | Open protocol (Coinbase-incubated, Linux Foundation-governed) using HTTP status 402: server replies with a price quote, client pays in stablecoin, retries with payment proof, gets the resource. |
| **Facilitator** | Hosted x402 service (we use Coinbase's) that verifies and settles payments so we never touch keys or run chain infrastructure. |
| **USDC** | A stablecoin: 1 USDC ≈ $1. What buyers pay with. |
| **Base / Base Sepolia** | Coinbase's Ethereum L2 network (mainnet) and its free test network. We accept payment on Base. |
| **Wallet address** | `0x…` identifier that receives funds; ours is a config value (`X402_PAYOUT_ADDRESS`). |
| **Supabase** | Managed Postgres + auth + admin UI ("Studio"). Our only datastore. |
| **RLS** | Row-Level Security: Postgres feature; we enable it in deny-all mode so only the server (service role) touches data. |
| **Ajv / Zod** | JSON-Schema validator (for schemas stored as data) / TypeScript validator (for internal contracts). |

**Import rules** (enforce with ESLint `no-restricted-imports`): `core` imports nothing internal; `learning`/`recipes`/`commerce` import only `core`; `hooks` imports `core`+`learning`; `generation` imports `core`+`learning`+`hooks`; `app/` imports modules; modules never import `app/` or components. ### 4.2 Key request flows

**Generation** — `POST /api/lab/runs` → executor loads generator row → renders prompt → calls LLM with output schema → per item: hook chain → insert entity + provenance → events. Response returns `run_id`; UI polls `GET /api/lab/runs/:id`.

**Curation** — cockpit `POST /api/lab/decisions` → route computes hashes (edits) → single Postgres function `decide_curation(...)` does snapshot + update + decision row + event atomically.

**Purchase** — agent `GET /api/v1/query/personality-profile-library?tags=trading` → middleware answers `402` with price quote → agent's x402 client pays USDC on Base via facilitator → retries with payment header → middleware verifies+settles → route: resolve recipe → whitelist filters → query → envelope with provenance block → record `x402_payments` + `query.served` event.

## 6. Shared conventions

**Canonical JSON & hashing.** The only hashed form of any content: keys sorted recursively, no whitespace, UTF-8, SHA-256 hex. One implementation (`core/canonical.ts`, §8.1); Postgres never computes hashes — routes pass them in.

**Error envelope.** Every non-2xx JSON response:

```json
{ "error": { "code": "invalid_params", "message": "human-readable", "details": {} } }
```

| code | HTTP | used for |
|---|---|---|
| `invalid_params` | 400 | Ajv/Zod validation failures (details = validator errors) |
| `unauthorized` | 401 | missing/invalid session on `/api/lab` |
| `payment_required` | 402 | emitted by x402 middleware (its own body format) |
| `forbidden` | 403 | valid session, email not in `ADMIN_EMAILS` |
| `not_found` | 404 | unknown product/run/entity |
| `rate_limited` | 429 | free endpoints (§13) |
| `generation_failed` | 502 | LLM provider error surfaced from executor |
| `internal` | 500 | everything else (log, don't leak) |

**Pagination.** Lab lists: `?limit=50&offset=0`, response `{items, total}`. Public query: `limit` capped by recipe `max_limit`; no offset (agents filter, not browse).

**Naming.** Tables/columns snake_case; event types `domain.action`; product/generator slugs kebab-case; TypeScript types PascalCase mirroring rows (`ProfileRow`, `GeneratorRow`).

**Timestamps** always `timestamptz`, UTC. **Money** as `NUMERIC`, USDC units. **IDs** UUID (`gen_random_uuid()`), except `events.id` BIGINT identity.

```sql
CREATE TABLE events (
  id          BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,

  actor_type  TEXT NOT NULL CHECK (actor_type IN ('human','system','hook','agent')),

CREATE RULE events_no_delete AS ON DELETE TO events DO INSTEAD NOTHING;

CREATE TABLE curation_decisions (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  entity_type      TEXT NOT NULL,
  entity_id        UUID NOT NULL,
  run_id           UUID,
  decision         TEXT NOT NULL
                   CHECK (decision IN ('approved','rejected','edited_approved')),
  reason_code      TEXT REFERENCES rejection_reasons(code),
  original_content JSONB NOT NULL,
  edited_content   JSONB,
  notes            TEXT,
  time_spent_ms    INT,
  judge_score      NUMERIC(3,2),
  judge_rubric     TEXT,
  decided_by       TEXT NOT NULL,
  created_at       TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_decisions_entity ON curation_decisions(entity_type, entity_id);
CREATE INDEX idx_decisions_run ON curation_decisions(run_id);

CREATE VIEW curation_queue AS
  SELECT 'profile' AS entity_type, id AS entity_id, content, quality_score,
         generation_run_id, created_at
  FROM profiles WHERE status = 'pending'
  ORDER BY created_at;

-- Atomic curation. Hashes are computed by the caller (JS) — Postgres never hashes.
CREATE OR REPLACE FUNCTION decide_curation(
  p_entity_type text, p_entity_id uuid, p_decision text,
  p_reason_code text, p_edited_content jsonb, p_new_content_sha256 char(64),
  p_notes text, p_time_spent_ms int,
  p_judge_score numeric, p_judge_rubric text, p_decided_by text
) RETURNS uuid LANGUAGE plpgsql SECURITY DEFINER AS $$
DECLARE
  v_original jsonb; v_version int; v_run uuid; v_decision_id uuid;
BEGIN
  IF p_entity_type <> 'profile' THEN
    RAISE EXCEPTION 'unsupported entity_type %', p_entity_type;  -- M2 extends
  END IF;
  SELECT content, version, generation_run_id INTO v_original, v_version, v_run
    FROM profiles WHERE id = p_entity_id AND status = 'pending' FOR UPDATE;
  IF NOT FOUND THEN RAISE EXCEPTION 'pending entity not found'; END IF;

  IF p_decision = 'approved' THEN
    UPDATE profiles SET status='approved' WHERE id=p_entity_id;
  ELSIF p_decision = 'rejected' THEN
    IF p_reason_code IS NULL THEN RAISE EXCEPTION 'reason_code required'; END IF;
    UPDATE profiles SET status='rejected' WHERE id=p_entity_id;
  ELSIF p_decision = 'edited_approved' THEN
    IF p_edited_content IS NULL OR p_new_content_sha256 IS NULL THEN
      RAISE EXCEPTION 'edited_content and hash required';
    END IF;
    UPDATE profiles SET
      content = p_edited_content,
      big_five = p_edited_content->'big_five',
      mbti_label = p_edited_content->>'mbti_label',
      decision_style = p_edited_content->>'decision_style',
      summary = p_edited_content->>'summary',
      tags = COALESCE((SELECT array_agg(x)
             FROM jsonb_array_elements_text(p_edited_content->'tags') x), tags),
      version = v_version + 1,
      status = 'approved'
    WHERE id = p_entity_id;
    INSERT INTO provenance (entity_type, entity_id, entity_version, model,
      prompt_hash, template_hash, params, sha256_content)
    VALUES (p_entity_type, p_entity_id, v_version+1, 'human-edit',
      repeat('0',64), repeat('0',64),
```

```ts
import { createHash } from 'crypto';

export function canonicalJson(value: unknown): string {
  if (Array.isArray(value)) return `[${value.map(canonicalJson).join(',')}]`;
  if (value && typeof value === 'object') {
    const keys = Object.keys(value as object).sort();
    return `{${keys.map(k =>
      `${JSON.stringify(k)}:${canonicalJson((value as Record<string, unknown>)[k])}`
    ).join(',')}}`;
  }
  return JSON.stringify(value);
}
```

```ts
import { canonicalJson, sha256 } from '@/modules/core/canonical';

export function renderTemplate(tpl: string, params: Record<string, unknown>): string {
  let out = tpl.replace(/\{\{#if (\w+)\}\}([\s\S]*?)\{\{\/if\}\}/g,
    (_, k, body) => {
      const v = params[k];
      return v !== undefined && v !== null && v !== '' && v !== 'none' ? body : '';
    });
  out = out.replace(/\{\{json (\w+)\}\}/g, (_, k) => canonicalJson(params[k] ?? null));
  out = out.replace(/\{\{(\w+)\}\}/g, (_, k) => String(params[k] ?? ''));
  return out;
}
```

```ts
    params: Record<string, unknown>; count: number; actorId: string }) {
  const { data: gen } = await dbAdmin.from('generators').select('*')
    .eq('slug', input.generatorSlug).eq('status', 'active')
    .order('version', { ascending: false }).limit(1).single();
  if (!gen) throw err('not_found', 404, `no active generator ${input.generatorSlug}`);

  const validate = ajv.compile(gen.params_schema);
  if (!validate(input.params))
    throw err('invalid_params', 400, 'params failed schema', validate.errors);

  const { data: run } = await dbAdmin.from('generation_runs').insert({

    params: input.params, model_used: gen.model_config.model,
    items_requested: input.count,
  }).select().single();

         actor_id: input.actorId, run_id: run.id,
         payload: { params: input.params, count: input.count } });

  // Serverless note: respond immediately with run.id; continue via
  // Vercel's waitUntil(after()) so the HTTP response isn't held open.
  processRun(gen, run, input).catch(() => {});
  return { runId: run.id };
}

async function processRun(gen: any, run: any,
    input: { params: Record<string, unknown>; count: number }) {
  try {
    // reserved system params
    const { data: biasRows } = await dbAdmin.from('biases').select('slug');
    const sys = { bias_slugs: (biasRows ?? []).map(b => b.slug) };
    let created = 0, cost = 0;
    for (const batch of batches(input.count, gen.model_config.max_items_per_call)) {
      const prompt = renderTemplate(gen.prompt_template,
        { ...input.params, ...sys, count: batch });
      const { items, costUsd } = await generateItems(gen.model_config, prompt,
        gen.output_schema);
      cost += costUsd;
      for (const item of items) {
        const results = await runHookChain(gen.hooks, { item, generator: gen, run });
        const verdict = finalVerdict(results);           // 'reject'|'pending'|'approve'
        if (verdict === 'reject') {
          await bump(run.id, 'items_rejected_by_hooks'); continue;
        }
        const entityId = await insertProfile(item, verdict, run.id);
        await dbAdmin.from('provenance').insert({
          entity_type: 'profile', entity_id: entityId, entity_version: 1,
          model: gen.model_config.model,
          prompt_hash: sha256(prompt), template_hash: templateHash(gen.prompt_template),
          params: { ...input.params, temperature: gen.model_config.temperature },
          sha256_content: contentHash(item),
        });
        emit({ event_type: 'generation.item_created', actor_type: 'system',
               entity_type: 'profile', entity_id: entityId, run_id: run.id,
               payload: { verdict } });
        created++;
      }
    }
    await dbAdmin.from('generation_runs').update({
      items_created: created, cost_usd: cost, status: 'done',
      finished_at: new Date().toISOString(),
    }).eq('id', run.id);
    emit({ event_type: 'generation.run_completed', actor_type: 'system',
           run_id: run.id, payload: { items_created: created, cost_usd: cost } });
  } catch (e) {
    await dbAdmin.from('generation_runs').update({
      status: 'failed', error: String(e), finished_at: new Date().toISOString(),
    }).eq('id', run.id);
    emit({ event_type: 'generation.run_failed', actor_type: 'system',
           run_id: run.id, payload: { error: String(e) } });
  }
}

async function insertProfile(item: any, verdict: string, runId: string) {
```

```ts
// registry.ts
import { emit } from '@/modules/learning/events';
import { schemaValidate } from './schema-validate';
import { dedup } from './dedup';
import { provenanceStamp } from './provenance';
import { route } from './route';
import type { Hook, HookContext, HookResult } from './types';

const REGISTRY: Record<string, Hook> = {
  schema_validate: schemaValidate, dedup, provenance_stamp: provenanceStamp, route,
};

export async function runHookChain(
  chain: Array<{ type: string; config?: Record<string, unknown> }>,
  ctx: HookContext,
): Promise<HookResult[]> {
  const results: HookResult[] = [];
  for (const step of chain) {
    const hook = REGISTRY[step.type];
    if (!hook) throw new Error(`unknown hook ${step.type}`);
    const r = await hook({ ...ctx, prior: results } as any, step.config ?? {});
    results.push(r);
    emit({ event_type: 'hook.executed', actor_type: 'hook', actor_id: step.type,
           run_id: ctx.run.id,
           payload: { hook: step.type, passed: r.passed, score: r.score ?? null } });
    if (r.verdict === 'reject') break;              // short-circuit
  }
  return results;
}
export function finalVerdict(results: HookResult[]): 'reject'|'pending'|'approve' {
  const routed = [...results].reverse().find(r => r.verdict);
  return routed?.verdict ?? 'pending';
}
```

```json
  "filters": { "status": "approved" },
  "allow_request_filters": ["tags","big_five_min","big_five_max","decision_style","mbti_label"],
  "default_limit": 20,
  "max_limit": 100
}
```

Request-filter semantics: `tags=trading,social` → array overlap; `big_five_min=openness:0.7,neuroticism:0.4` (and `_max`) → per-trait numeric bounds; `decision_style`/`mbti_label` → exact match; `limit` → capped by `max_limit`. Unknown params are **ignored, never errors** (agents probe) — but logged in the `query.*` event payload for demand analysis.

```ts
// resolver.ts
import { dbAdmin } from '@/modules/core/db';
import { err } from '@/modules/core/errors';

export async function resolveQuery(rules: any, req: URLSearchParams) {
  if (rules.entity !== 'profile') throw err('internal', 500, 'unsupported entity'); // M2 extends
  let q = dbAdmin.from('profiles')
    .select('id, version, big_five, mbti_label, decision_style, summary, tags')
    .eq('status', rules.filters.status ?? 'approved');

  const allowed = new Set(rules.allow_request_filters ?? []);
  if (allowed.has('tags') && req.get('tags'))
    q = q.overlaps('tags', req.get('tags')!.split(',').map(s => s.trim()));
  if (allowed.has('decision_style') && req.get('decision_style'))
    q = q.eq('decision_style', req.get('decision_style'));
  if (allowed.has('mbti_label') && req.get('mbti_label'))
    q = q.eq('mbti_label', req.get('mbti_label')!.toUpperCase());
  for (const dir of ['min', 'max'] as const) {
    const raw = req.get(`big_five_${dir}`);
    if (!allowed.has(`big_five_${dir}`) || !raw) continue;
    for (const pair of raw.split(',')) {
      const [trait, v] = pair.split(':');
      if (!['openness','conscientiousness','extraversion','agreeableness','neuroticism']
           .includes(trait)) continue;                    // ignore unknown
      const num = Number(v);
      if (Number.isNaN(num)) continue;
      q = dir === 'min' ? q.gte(trait, num) : q.lte(trait, num);
      // trait names are real generated columns (§7.1) — correct numeric
      // comparison and indexable; never compare via JSONB ->> text
    }
  }
  const limit = Math.min(Number(req.get('limit')) || rules.default_limit, rules.max_limit);
  const { data, error } = await q.limit(limit);
  if (error) throw err('internal', 500, error.message);
  return data ?? [];
}
```

```ts
// app/api/lab/decisions/route.ts
import { z } from 'zod';
import { decide } from '@/modules/learning/decisions';
import { requireAdmin } from '@/modules/core/auth';   // session + allowlist check
import { toResponse } from '@/modules/core/errors';

const Body = z.object({
  entity_type: z.literal('profile'),
  entity_id: z.string().uuid(),
  decision: z.enum(['approved', 'rejected', 'edited_approved']),
  reason_code: z.string().optional(),
  edited_content: z.unknown().optional(),
  notes: z.string().optional(),
  time_spent_ms: z.number().int().nonnegative().optional(),
}).refine(b => b.decision !== 'rejected' || !!b.reason_code,
          { message: 'reason_code required when rejecting' })
  .refine(b => b.reason_code !== 'other' || !!b.notes,
          { message: 'notes required for reason \'other\'' })
  .refine(b => b.decision !== 'edited_approved' || b.edited_content !== undefined,
          { message: 'edited_content required for edits' });

export async function POST(req: Request) {
  try {
    const admin = await requireAdmin(req);
    const b = Body.parse(await req.json());
    const id = await decide({
      entityType: b.entity_type, entityId: b.entity_id, decision: b.decision,
      reasonCode: b.reason_code, editedContent: b.edited_content,
      notes: b.notes, timeSpentMs: b.time_spent_ms, decidedBy: admin.email,
    });
    return Response.json({ decision_id: id });
  } catch (e) { return toResponse(e); }
}
```

```tsx
// components/lab/SchemaForm.tsx
'use client';
export function SchemaForm({ schema, value, onChange }: {
  schema: any; value: Record<string, any>;
  onChange: (v: Record<string, any>) => void;
}) {
  const props = schema.properties ?? {};
  const set = (k: string, v: any) => onChange({ ...value, [k]: v });
  return (
    <div className="space-y-3">
      {Object.entries<any>(props).map(([key, p]) => (
        <label key={key} className="block">
          <span className="text-sm font-medium">{key}
            {schema.required?.includes(key) && ' *'}</span>
          {p.enum ? (
            <select className="input" value={value[key] ?? p.default ?? ''}
                    onChange={e => set(key, e.target.value)}>
              {p.enum.map((o: string) => <option key={o}>{o}</option>)}
            </select>
          ) : p.type === 'boolean' ? (
            <input type="checkbox" checked={value[key] ?? p.default ?? false}
                   onChange={e => set(key, e.target.checked)} />
          ) : p.type === 'integer' || p.type === 'number' ? (
            <input type="number" className="input" min={p.minimum} max={p.maximum}
                   value={value[key] ?? p.default ?? ''}
                   onChange={e => set(key, p.type === 'integer'
                     ? parseInt(e.target.value) : parseFloat(e.target.value))} />
          ) : p.type === 'array' ? (
            <input className="input" placeholder="comma-separated"
                   value={(value[key] ?? p.default ?? []).join(',')}
                   onChange={e => set(key, e.target.value.split(',')
                     .map(s => s.trim()).filter(Boolean))} />
          ) : (
            <input className="input" value={value[key] ?? p.default ?? ''}
                   onChange={e => set(key, e.target.value)} />
          )}
          {p.description && <span className="text-xs text-gray-500">{p.description}</span>}
        </label>
      ))}
    </div>
  );
}
```

## 16. Runbook (operations)

- **Health**: `/api/health` → `{ok, db_ms, version}` (DB ping + git SHA). Uptime monitor (any pinger) on it + on `/api/v1/products`.
- **Stuck run** (`running` > 15 min): serverless died mid-run. `UPDATE generation_runs SET status='failed', error='stalled' WHERE status='running' AND created_at < now()-interval '15 min';` Items already inserted are safe (each item commits independently); just re-run the remainder.
- **LLM provider outage**: runs fail with `generation_failed`. Switch provider = new generator version with different `model_config` (2 minutes in Studio in M1, in the Lab from M3), or wait.
- **Facilitator/x402 outage**: query routes will 402-loop or error. Acceptable M1 response: post status on `/docs`; no code change needed (buyers retry). Escalate only if > 1 h.
- **Backups**: Supabase PITR on Pro plan; additionally nightly `pg_dump` via GitHub Action to private storage once revenue is real. Restore drill: once, into staging, before mainnet cutover.
- **Key rotation**: model keys and CDP keys are env swaps + redeploy; payout address change requires middleware env change + `/docs` note.
- **Weekly lab review** (owner, 30 min, from `/lab/stats`): top rejection reason per generator version → prompt fix as new version; cost-per-approved trend; demand report → next runs. This cadence *is* the learning loop until M3 automates it.
